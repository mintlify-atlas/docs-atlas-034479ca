---
title: "Mesh offsetting"
description: "Generate offset surfaces for 3D printing and machining"
---

Mesh offsetting creates a new surface displaced by a fixed distance from the input mesh. Positive offsets expand the mesh outward; negative offsets shrink it inward. Common applications include generating tool-path clearance volumes for CNC machining, adding shell thickness for 3D printing, and computing Minkowski sums.

MeshLib implements offset through a signed distance field (voxel) approach. The input mesh is converted to a voxel grid, offset is applied in voxel space, and the result is converted back to a mesh via marching cubes.

## OffsetParameters

| Field | Type | Description |
|-------|------|-------------|
| `voxelSize` | `float` | Size of each voxel cell. Smaller values produce a more accurate offset surface at the cost of higher memory and computation. |
| `signDetectionMode` | `SignDetectionMode` | Controls how the interior/exterior of the mesh is determined. Set to `HoleWindingRule` when the mesh has open boundaries. |

## Code examples

<CodeGroup>

```python offset.py
import meshlib.mrmeshpy as mrmeshpy
import sys

# Load mesh
mesh = mrmeshpy.loadMesh("mesh.stl")

# Setup parameters
params = mrmeshpy.OffsetParameters()
params.voxelSize = mesh.computeBoundingBox().diagonal() * 5e-3 # offset grid precision (algorithm is voxel based)
if mrmeshpy.findRightBoundary(mesh.topology).empty():
    params.signDetectionMode = mrmeshpy.SignDetectionMode.HoleWindingRule  # use if you have holes in mesh

# Make offset mesh
offset = mesh.computeBoundingBox().diagonal() * 0.05
try:
    result_mesh = mrmeshpy.offsetMesh(mesh, offset, params)
except ValueError as e:
    print(e)
    sys.exit(1)

# Save result
mrmeshpy.saveMesh(result_mesh, "offsetMesh.stl")
```

```cpp offset.cpp
#include <MRMesh/MRBox.h>
#include <MRMesh/MRMesh.h>
#include <MRMesh/MRCube.h>
#include <MRMesh/MRMeshSave.h>
#include <MRMesh/MRRegionBoundary.h>
#include <MRVoxels/MROffset.h>

#include <iostream>

int main()
{
    // Create mesh
    MR::Mesh mesh = MR::makeCube();

    // Setup parameters
    MR::GeneralOffsetParameters params;
    // calculate voxel size depending on desired accuracy and/or memory consumption
    params.voxelSize = suggestVoxelSize( mesh, 10000000.f );
    if ( !MR::findRightBoundary( mesh.topology ).empty() )
        params.signDetectionMode = MR::SignDetectionMode::HoleWindingRule; // use if you have holes in mesh

    // Make offset mesh
    float offset = mesh.computeBoundingBox().diagonal() * 0.1f;
    auto meshRes = MR::generalOffsetMesh( mesh, offset, params );

    if ( !meshRes.has_value() )
    {
        std::cerr << meshRes.error() << std::endl;
        return 1;
    }

    // Save result
    if ( auto saveRes = MR::MeshSave::toAnySupportedFormat( *meshRes, "mesh_offset.stl" ); !saveRes )
    {
        std::cerr << saveRes.error() << std::endl;
        return 1;
    }

    return 0;
}
```

</CodeGroup>

<Tip>
Choose `voxelSize` as a fraction of the mesh's bounding box diagonal — `diagonal * 5e-3` is a reasonable default that balances accuracy against memory. Halving `voxelSize` increases memory usage by roughly 8x. Use `suggestVoxelSize(mesh, targetVoxelCount)` in C++ to pick a size that stays within a target voxel budget.
</Tip>
