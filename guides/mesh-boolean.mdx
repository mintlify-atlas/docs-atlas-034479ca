title: "Mesh boolean operations"
description: "Perform CSG union, intersection, and difference on 3D meshes"

Boolean operations — also called Constructive Solid Geometry (CSG) — combine two meshes into one by computing the surface that results from a set-theoretic operation on their volumes. Common uses include subtracting a channel from a body, unioning parts into a single shell, or computing the intersection volume of two overlapping objects.

## BooleanOperation enum

The `BooleanOperation` enum controls which portion of the two meshes is kept in the result.

| Value | Description |
|-------|-------------|
| `InsideA` | Part of mesh A that is inside mesh B |
| `InsideB` | Part of mesh B that is inside mesh A |
| `OutsideA` | Part of mesh A that is outside mesh B |
| `OutsideB` | Part of mesh B that is outside mesh A |
| `Union` | All geometry inside mesh A **or** mesh B (A ∪ B) — outside parts of both |
| `Intersection` | Only geometry inside **both** A and B (A ∩ B) — inside parts of both |
| `DifferenceAB` | Surface of A minus surface of B (outside A, inside B) |
| `DifferenceBA` | Surface of B minus surface of A (outside B, inside A) |

## Code examples

<CodeGroup>

```python boolean.py
import meshlib.mrmeshpy as mrmeshpy

# create first sphere with radius of 1 unit
sphere1 = mrmeshpy.makeUVSphere(1.0, 64, 64)

# create second sphere by cloning the first sphere and moving it in X direction
sphere2 = mrmeshpy.copyMesh(sphere1)
xf = mrmeshpy.AffineXf3f.translation(mrmeshpy.Vector3f(0.7, 0.0, 0.0))
sphere2.transform(xf)

# perform boolean operation
result = mrmeshpy.boolean(sphere1, sphere2, mrmeshpy.BooleanOperation.Intersection)
if not result.valid():
    print(result.errorString)
else:
    # save result to STL file
    mrmeshpy.saveMesh(result.mesh, "out_boolean.stl")
```

```cpp boolean.cpp
#include <MRMesh/MRMesh.h>
#include <MRMesh/MRMeshBoolean.h>
#include <MRMesh/MRMeshSave.h>
#include <MRMesh/MRUVSphere.h>

#include <iostream>

int main()
{
    // create first sphere with radius of 1 unit
    MR::Mesh sphere1 = MR::makeUVSphere( 1.0f, 64, 64 );

    // create second sphere by cloning the first sphere and moving it in X direction
    MR::Mesh sphere2 = sphere1;
    MR::AffineXf3f xf = MR::AffineXf3f::translation( MR::Vector3f( 0.7f, 0.0f, 0.0f ) );
    sphere2.transform( xf );

    // perform boolean operation
    MR::BooleanResult result = MR::boolean( sphere1, sphere2, MR::BooleanOperation::Intersection );
    if ( !result.valid() )
        std::cerr << result.errorString << std::endl;

    MR::Mesh resultMesh = *result;

    // save result to STL file
    if ( auto saveRes = MR::MeshSave::toAnySupportedFormat( resultMesh, "out_boolean.stl" ); !saveRes )
    {
        std::cerr << saveRes.error() << std::endl;
        return 1;
    }

    return 0;
}
```

</CodeGroup>

## BooleanResult

`boolean()` returns a `BooleanResult` rather than throwing on failure. Always check `valid()` before accessing the output mesh.

| Member | Type | Description |
|--------|------|-------------|
| `mesh` | `Mesh` | The output mesh when the operation succeeds |
| `errorString` | `string` | Human-readable description of the failure cause; empty on success |
| `valid()` | `bool` | Returns `true` when `errorString` is empty and `mesh` is usable |

In C++, `BooleanResult` supports `operator*` so you can write `*result` to access the mesh directly after a successful check.

<Note>
Input meshes must have no self-intersections in the zone where the two meshes intersect each other. Self-intersections elsewhere in either mesh are tolerated, but any within the intersecting zone will cause `valid()` to return `false`.
</Note>

<Warning>
Boolean operations require both input meshes to be closed (watertight). Open meshes — those with boundary edges — prevent MeshLib from determining which side is interior, causing `Union`, `Intersection`, and `Difference` to fail or produce unexpected output. Use [mesh repair](/guides/mesh-repair) to fill holes before calling `boolean()`.
</Warning>
